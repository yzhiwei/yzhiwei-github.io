写于博客建成之后

发表于 2019-01-24
字数统计: 745 | 阅读时长 ≈ 2

为什么会选择搭一个博客了

搭这个博客的初衷一方面是觉得搭一个博客挺好玩的，另一方面也是希望自己能够记录一些东西，用输出来促进输入。不断地学习才应该是一种正确的生活方式嘛，如果博客中记录的东西能够帮助到他人，那么就变得更完美了。

搭博客的心路历程

自己看了一个学长的教程，他在教程中介绍了3种方法，一种不好配置，一种就是他采用的，另外一种就是付费的，学长说ruby不好配置，所以我果断的选择了ruby，但一开始的软件安装就出现了问题，环境始终配置不对，所以果断放弃。老老实实的采用了github+hexo来搭建我的博客。截至到写这篇文章为止，博客也还没有完全配置好，只能说是勉强能看了吧。每一步都是坑，没有一步是能够直接到位的。还是挺感谢csdn的，每一步出错都能够在csdn里面找到解决方法。

部分关键性步骤

注册github账号

进入GitHub官网，注册一个github账号，弄一个仓库（这应该是整个流程中最简单的一步了）

必要的软件

git node.js
由于国内一些原因，总会有一些不好翻越的墙，会导致下载速度变得奇慢，在后续步骤中下载一些插件速度也比较慢，在机房百兆网络下，看到下载速度1kbs,内心还是非常绝望的，但这在神奇的csdn中，都不是问题，无意中发现了一个淘宝分流的方法，会让速度变得正常。

hexo

这一步用命令行安装就好

配置git

在git中输出user name和keyword

密钥

运行ssh，生成公钥和私钥，通过公钥和私钥的配对来确定自己的身份

主题的安装

看到了很多主题，最终选择了next，这个主题看起来比较清爽，符合程序员的特质
这一部分不是最难的，但肯定可以成为最繁琐的，每一个功能，每一个小细节，都需要你去配置。

遇到的一些坑

最让人绝望的是莫过于你本地都配置好了，却在网页上面显示404，乱码，主题只显示框架。这三种情况我都遇到过，真的特别让人崩溃。

写这篇文章时的准备

这个与普通的文本输入有一定差别，首先在建文章时，是通过cmd敲hexo命令来实现的，文件格式为.md。它采用了markdown语法排版，与office排版方法还是有很大区别的。

写在最后

希望这一片文章不是我在博客上面更的最后一篇文章就好。
